# 41. Virtual Partner Reviewer Bot (【査読支援】仮想パートナーの多角レビューボット)

⭐ **If you find this workflow helpful, please give it a Star! (役に立ったらぜひスターをお願いします！)** ⭐

## 🚀 概要 (Overview)
一人エンジニアや少人数チーム特有の「客観的な視点でレビュー（壁打ち）してほしい」という悩みを解決する、多角的なAI査読ボットです。

Slackの専用チャンネル（例: `#壁打ち・査読`）に、書きかけの企画書、クライアントへの提案メール、コードの設計メモなどを投稿するだけで、AI（Gemini）が以下の**3つの異なるペルソナ（役職）**に扮して、多角的なフィードバックを瞬時に返してくれます。

1. **💻 シニアエンジニアの視点:** 技術的な実現性、エッジケースの抜け漏れ、セキュリティリスクの指摘。
2. **👔 決裁者・クライアントの視点:** ビジネス上のメリット（ROI）、目的の明確さ、説得力へのツッコミ。
3. **📝 編集者の視点:** 文章の構成、専門用語の多用、トーン＆マナーの改善提案。

さらに、孤独な作業を労う「温かい励ましの一言」も添えてくれる、頼れる仮想の右腕です。

## ✨ 特徴 (Features)
* **多角的なプロンプト設計:** 1回のプロンプトで3つの明確な評価軸（技術、ビジネス、構成）を持たせることで、偏りのない網羅的なレビューを実現しています。
* **Slackスレッドへの自動返信:** n8nの機能（`threadTs`の指定）を活用し、元の投稿に対して「スレッド」として結果をぶら下げるため、チャンネルのタイムラインが長文で荒れるのを防ぎます。
* **心理的安全性への配慮:** 単なる厳しい指摘だけでなく、最後に必ず「ポジティブな励まし」を出力するよう設計されており、ユーザーのモチベーションを維持します。

## ⚙️ 使用しているノード (Nodes Used)
1. **Slack Trigger:** 専用チャンネルへのメッセージ投稿を検知。
2. **Google Gemini:** 投稿されたテキストを読み込み、3つのペルソナによる査読結果をJSONで出力。
3. **Code (JSONパース):** Geminiの出力結果と、元のSlack投稿のタイムスタンプ（`ts`）を結合。
4. **Slack (Reply):** 整形された査読レポートを、元の投稿へのスレッド返信として送信。

## 🛠️ 使い方 (How to Use)
1. このディレクトリにある `workflow.json` をご自身のn8n環境にインポートします。
2. 各ノードの認証情報を設定します（Slack, Gemini）。
3. **Slack Trigger** ノードで、監視するチャンネル（例: `#壁打ち・査読`）を指定します。
4. **Slack (Reply)** ノードでも、同じチャンネルを指定します。
5. ワークフローをActiveにし、該当チャンネルに提案書やコードのメモなどを投稿して動作を確認してください。
