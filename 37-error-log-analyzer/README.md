# 37. Error Log AI Analyzer (エラーログ自動翻訳＆解決策提示ボット)

⭐ **If you find this workflow helpful, please give it a Star! (役に立ったらぜひスターをお願いします！)** ⭐

## 🚀 概要 (Overview)
システム障害発生時に飛んでくる「難解な英語のエラーログ」や「長大なスタックトレース」を、AI（Gemini）が瞬時に解析し、分かりやすい日本語でSlackへ通知するSRE（サイト信頼性エンジニア）支援ワークフローです。

Webhook経由でエラーログを受け取ると、Geminiが経験豊富なエンジニアとして振る舞い、「エラーの概要」「推測される原因」「すぐに実行すべき3つの初動対応」をセットにしてアラートを鳴らします。若手エンジニアや非エンジニアの情シス担当者でも、パニックにならず迅速な一次対応が可能になります。

## ✨ 特徴 (Features)
* **Webhookによる圧倒的な汎用性:** AWS、GCP、Datadog、Sentry、自社開発アプリなど、Webhook（POSTリクエスト）を送信できるあらゆる監視システムと連携可能です。
* **SREレベルのインサイト:** 単なる翻訳ではなく、「おそらくDBの接続プールが枯渇しています。まずは再起動し、コネクション数の設定を見直してください」といった具体的なアクションを引き出します。
* **JSONの丸投げ解析:** 複雑なJSON構造のエラーログでも、そのまま文字列化（`JSON.stringify`）してGeminiに投げるだけで、AIが必要なコンテキストを自ら抽出して読み解きます。

## ⚙️ 使用しているノード (Nodes Used)
1. **Webhook:** 外部システムからのエラー通知（POSTリクエスト）を受け取るトリガーノード。
2. **Google Gemini:** 受け取ったエラーログのJSONデータを解析し、日本語のレポートを生成。
3. **Slack:** 生成されたアラートと解決策を、開発チームや情シスのチャンネルへ通知。

## 🛠️ 使い方 (How to Use)
1. このディレクトリにある `workflow.json` をご自身のn8n環境にインポートします。
2. 各ノードの認証情報を設定します。
   * Google Gemini API キー
   * Slack アカウント
3. **Webhookノード** を開き、「Test URL」をコピーします。
4. **Slackノード** の `Channel` に、通知を送りたい開発用チャンネルIDを入力します。
5. ワークフローを「Listen for test event（テスト実行）」状態にし、Postmanやターミナル（curlコマンド）からコピーしたTest URL宛に、適当なエラーJSONをPOST送信して動作を確認してください。
