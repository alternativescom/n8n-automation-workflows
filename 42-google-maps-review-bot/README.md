# 42. Google Maps Review Response & Anti-Flame Bot (【店舗支援】Googleマップ口コミ対応＆炎上防止ボット)

⭐ **If you find this workflow helpful, please give it a Star! (役に立ったらぜひスターをお願いします！)** ⭐

## 🚀 概要 (Overview)
飲食店やサロンなど、実店舗ビジネス（ローカルビジネス）の評価を左右する「Googleマップの口コミ」に対する初動対応を自動化・効率化するワークフローです。

新しい口コミが投稿されると、AI（Gemini）が優秀なPR担当者としてレビュー内容の感情分析（ポジティブ／ネガティブ）を行います。
高評価には再来店を促す温かいメッセージを、低評価には炎上を防ぐための誠実な謝罪と事実確認を促す「神対応」の返信ドラフトを自動生成し、店長や責任者に即座に通知します。

## ✨ 特徴 (Features)
* **感情認識と自動分岐（Switch）:** AIが口コミのトーンを判定し、高評価と低評価で異なる通知ルートへ処理を正確に振り分けます。
* **プロフェッショナルな返信ドラフト:** 忙しい現場のスタッフがゼロから文章を考える手間を省き、そのままコピペして使える品質の返信文をAIが生成します。
* **📱 LINE公式アカウントへの簡単な切り替え（推奨）:** 本テンプレートは汎用性を考慮して「Slack通知」で作成していますが、**通知先を「LINE」に置き換えることが簡単に可能です。** パソコンやSlackを使わない店舗スタッフ向けに、最後のSlackノードをLINE Messaging API（HTTP Requestノード等）に差し替えるだけで、店長のスマホ（LINE）へダイレクトに返信文案を届ける、極めて実用的なシステムへと進化します。

## ⚙️ 使用しているノード (Nodes Used)
1. **Webhook:** Googleビジネスプロフィール等からの新しい口コミデータを受信。
2. **Google Gemini:** 口コミ内容を分析し、感情（POSITIVE/NEGATIVE）の判定と返信文案を生成。
3. **Code (JSONパース):** AIの出力結果を後続のルーティング用にパース。
4. **Switch:** 感情判定の結果をもとに、高評価ルートと低評価ルートへ分岐。
5. **Slack:** 店舗マネージャーへ、口コミ内容と返信ドラフトを通知（※LINEへの変更も容易です）。

## 🛠️ 使い方 (How to Use)
1. このディレクトリにある `workflow.json` をご自身のn8n環境にインポートします。
2. 各ノードの認証情報を設定します（Gemini, Slack）。
3. **Slack** ノードで、通知先のチャンネルを指定します。
4. ワークフローをテスト状態（Listen for test event）にし、Webhook宛てにテスト用の口コミデータ（JSON）をPOSTして動作を確認してください。
