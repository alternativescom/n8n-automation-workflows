# 40. Web Designer AI Screening Bot (【採用支援】Webデザイナー特化型 AIスクリーニングボット)

⭐ **If you find this workflow helpful, please give it a Star! (役に立ったらぜひスターをお願いします！)** ⭐

## 🚀 概要 (Overview)
Web制作会社や事業会社のデザイン部門に向けた、**「Webデザイナー特化型」**の採用一次スクリーニング自動化ワークフローです。

応募フォーム（Webhook）から送られてきた自己PRや使用ツール、ポートフォリオのコンセプト文を、AI（Gemini）が**「厳格かつ優秀なシニアアートディレクター」**として評価します。
単なるキーワードマッチングではなく、「Figmaのコンポーネント設計ができるか」「表層的な絵作りだけでなく、UI/UXや実装（コーディング）への理解があるか」といった、現場のリアルな評価軸で100点満点でスコアリングします。

評価結果は、「強み」や「面接で深掘りすべき懸念点」と共に採用担当者のSlackへ即座に通知され、書類選考のスピードアップと面接の質向上を同時に実現します。

## ✨ 特徴 (Features)
* **プロフェッショナルな評価軸:** Geminiに「シニアアートディレクター」のペルソナを与え、Web業界特有の専門的な視点（UI/UX視点、エンジニア連携など）でポテンシャルを見抜きます。
* **面接の「質問カンペ」を自動生成:** 単に合否の目安を出すだけでなく、「面接での確認事項（懸念点）」を具体的に提示するため、面接官（現場マネージャー等）の事前準備の負担が激減します。
* **迅速な初動対応:** 応募があった瞬間に一次評価がSlackに届くため、優秀な人材（A評価）に対するアプローチの遅れ（機会損失）を防ぎます。

## ⚙️ 使用しているノード (Nodes Used)
1. **Webhook:** 採用サイトやGoogleフォームからのエントリー情報を受信。
2. **Google Gemini:** 応募内容を読み込み、アートディレクターとして評価・採点し、JSON形式でレポートを生成。
3. **Code (JSONパース):** Geminiの出力結果と、元の応募者情報（氏名やURLなど）を綺麗に結合。
4. **Slack:** 採用担当者や現場マネージャーのチャンネルへ、整形された評価レポートを通知。

## 🛠️ 使い方 (How to Use)
1. このディレクトリにある `workflow.json` をご自身のn8n環境にインポートします。
2. 各ノードの認証情報を設定します（Gemini, Slack）。
3. **Slack** ノードで、通知先のチャンネル（例: `#採用チーム`）を指定します。
4. ワークフローを「Listen for test event」にし、Postmanやターミナル（curl）からWebhook URL宛てにテスト用の応募データ（JSON）をPOST送信して動作を確認してください。
