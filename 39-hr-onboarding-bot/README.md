# 39. HR Onboarding & Account Generation Bot (【HR支援】新入社員オンボーディング自動化ボット)

⭐ **If you find this workflow helpful, please give it a Star! (役に立ったらぜひスターをお願いします！)** ⭐

## 🚀 概要 (Overview)
4月の入社シーズンに人事・総務部門を悩ませる「アカウント発行」と「オンボーディング連絡」の手間をゼロにする、HR特化型の自動化ワークフローです。

人事担当者が新入社員のデータ（氏名、配属部署、個人の連絡先など）をフォームから送信するだけで、以下の処理が全自動で実行されます。
1. **アカウントの自動生成:** 氏名から社用メールアドレスと初期パスワードを自動生成します。（※本テンプレートではCodeノードを用いた模擬処理になっています）
2. **パーソナライズされた歓迎メール:** AI（Gemini）が配属部署や役割に合わせた「温かみのある歓迎メッセージ」を作成し、アカウント情報と共に個人のメールアドレスへ送信します。
3. **全社向けSlack紹介:** 同時に、Slackの全社チャンネル（#general等）へ、絵文字を交えたワクワクする新入社員の紹介文を自動投稿し、歓迎ムードを作ります。

手作業による設定ミスや連絡漏れを防ぎ、新入社員が「入社初日からスムーズに、かつ歓迎されていると感じながら」業務をスタートできる環境を構築します。

## ✨ 特徴 (Features)
* **事務作業と感情ケアの両立:** 無機質なアカウント通知だけでなく、Geminiを活用して「配属部署」や「役割」に応じたパーソナライズされた歓迎文を添えることで、従業員エンゲージメント（EX）を高めます。
* **並行処理によるタイムラグの排除:** メールの送信とSlackへの紹介投稿を完全に同時に行うため、情報の時差が発生しません。
* **拡張性の高いモック設計:** アカウント生成部分はあえてCodeノードで模擬処理（モック）にしています。実運用時は、ここを「Google Workspace」や「Microsoft Entra ID」のノードに差し替えるだけで、実際のアカウント発行処理にシームレスに移行できます。

## ⚙️ 使用しているノード (Nodes Used)
1. **Webhook:** 人事データベースやGoogleフォーム等からの入社データを受信。
2. **Code (アカウント生成):** 氏名のローマ字からメールアドレスとパスワードを自動生成。
3. **Google Gemini:** 配属部署や役割に応じた歓迎メール本文と、Slack用の紹介文を生成。
4. **Code (JSONパース):** Geminiの生成テキストとアカウント情報を結合。
5. **Gmail:** 新入社員の個人メールアドレス宛てに案内を送信。
6. **Slack:** 全社チャンネルへ新入社員の紹介メッセージを投稿。

## 🛠️ 使い方 (How to Use)
1. このディレクトリにある `workflow.json` をご自身のn8n環境にインポートします。
2. 各ノードの認証情報を設定します（Gemini, Gmail, Slack）。
3. **Slack** ノードで、通知先のチャンネル（例: `#general`）を指定します。
4. ワークフローをActive（またはテスト待機状態）にし、WebhookのURLに対して新入社員のJSONデータをPOST送信して動作を確認してください。
