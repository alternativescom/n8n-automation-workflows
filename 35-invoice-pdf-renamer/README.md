# 35. Invoice PDF Renamer (電帳法対応：請求書PDFリネーム＆自動保存ボット)

## 🚀 概要 (Overview)
メールで届いたPDF形式の請求書をAI（Gemini 2.5 Flash）のマルチモーダル機能で読み取り、電子帳簿保存法（電帳法）の検索要件に合わせたファイル名（`取引年月日_取引先名_取引金額.pdf`）に自動でリネームして、指定のGoogleドライブへ保存するn8nワークフローです。

経理担当者の「手作業でPDFを開き、金額を確認して、ファイル名を手打ちで変更する」という、退屈でミスの起きやすいルーティンワークを完全にゼロにします。

## ✨ 特徴 (Features)
* **マルチモーダル解析:** Geminiの画像・PDF解析能力を活用し、フォーマットがバラバラな各社の請求書から正確に「日付・会社名・金額」を抽出します。
* **電帳法（電子帳簿保存法）対応:** 抽出したデータを元に、法要件を満たすファイル名ルールをCodeノードで安全に自動生成します。
  * 変換例：`20260315_株式会社〇〇_150000.pdf`
* **シームレスな自動化連携:** Gmailでの受信から、Googleドライブへの保存、Slackへの完了通知までを人間の手を介さずに全自動で行います。

## ⚙️ 使用しているノード (Nodes Used)
1. **Gmail Trigger:** 添付ファイル（PDF）付きのメールを受信したタイミングで起動。
2. **Google Gemini:** 添付されたPDFファイルを解析し、必要な情報をJSON形式で抽出。
3. **Code:** Geminiが出力したデータをパースし、新しいファイル名（文字列）を生成。
4. **Google Drive:** 取得した元のPDFファイルを、新しいファイル名で指定のフォルダへアップロード。
5. **Slack:** 処理が完了したことを経理チームのチャンネルへ通知。

## 🛠️ 使い方 (How to Use)
1. このディレクトリにある `workflow.json` をダウンロードし、ご自身のn8n環境にインポートします。
2. 各ノードの認証情報（Credentials）を設定します。
   * Gmail アカウント
   * Google Gemini API キー
   * Google Drive アカウント
   * Slack アカウント
3. **Google Driveノード**の `Folder ID` に、保存先となるご自身のドライブのフォルダIDを入力してください。
4. **Slackノード**の `Channel` に、通知を送りたいチャンネルIDを入力してください。
5. ワークフローを「Active」にしてテストメールを送信してみてください。
