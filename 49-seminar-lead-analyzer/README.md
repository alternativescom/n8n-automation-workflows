# 49. Seminar Lead Analyzer (セミナー参加者の「熱量分析＆個別提案メール」自動生成ボット)

⭐ **If you find this workflow helpful, please give it a Star! (役に立ったらぜひスターをお願いします！)** ⭐

## 🚀 概要 (Overview)
「セミナーを開いて満足」で終わらせないための、強力なインサイドセールス自動化ボットです。
セミナー参加者が入力したアンケート結果をAI（Gemini）が即座に分析。「この企業は今、どのくらいDXに本気か（熱量）」をスコアリングし、相手の課題に直接刺さる「個別フォローアップメールの文面」を自動生成してSlackに通知します。

全員に同じ「本日はありがとうございました」という定型文を送る時代は終わりました。記憶が新しいうちに、超パーソナライズされた提案を届けることで、コンサルティングやシステム開発の受注率を劇的に引き上げます。

## ✨ 特徴 (Features)
* **自動スコアリング:** アンケートの自由記述から、顧客のペイン（悩み）の深さと受注確度をAIが数値化。
* **パーソナライズされた提案:** 「IT人材がいない」「コストをかけたくない」といった相手の個別事情を汲み取った、誠実で自然な営業メールをドラフトします。
* **ネクストアクションの提示:** AIが「まずは小さな自動化の成功体験を提案しましょう」など、営業担当者が次に取るべき具体的な戦略までアドバイスします。

## ⚙️ 使用しているノード (Nodes Used)
* **Manual Trigger / Set:** テスト実行・ダミーデータ生成用（※実運用時は削除します）
* **Google Gemini:** アンケート分析、スコアリング、メール文面生成
* **Code:** JSONのパース処理
* **Slack:** 営業チーム（自分）へのリード情報とドラフトの通知

## 🛠️ 実運用環境への移行手順（⚠️ ここが一番重要です！）

インポート直後のワークフローは「テスト用」にダミーデータがセットされています。
実際のセミナーアンケート（Googleフォームなど）と連携して実運用レベルにするには、以下の手順でノードを置き換えてください。

### Step 1: トリガー（起点）を実際のフォームに置き換える
一番左にある `Manual Trigger` ノードを削除し、あなたが使っているアンケートツールに合わせて新しいトリガーを配置します。

* **Googleフォームを使っている場合:**
  `Google Forms Trigger` ノードを追加し、認証（Credential）を行って対象のフォームを選択します。
* **Tally, Typeformなどの最新ツールを使っている場合:**
  `Webhook` ノードを追加（メソッドはPOST）。発行されたWebhook URLを、お使いのフォームツールの設定画面に貼り付けます。

### Step 2: ダミーデータを削除し、本物のデータ（変数）を紐付ける
1. テスト用の `Set (ダミーアンケート回答)` ノードは**削除**します。これでもう固定のデータは流れなくなります。
2. 新しく配置したトリガーノード（Google FormsやWebhook）と、`Gemini` ノードを線で繋ぎます。
3. `Gemini` ノードを開き、プロパティの「Messages」内にある文章を修正します。
   * 例えば `企業名: {{ $json.company_name }}` となっている部分を消し、**左側の入力データ一覧から、実際のフォームの「会社名」の項目をドラッグ＆ドロップ**して置き換えます。氏名やアンケート回答内容（自由記述欄）も同様に、本物のデータに紐付け直してください。

### Step 3: Slack通知先の設定
最後尾の `Slack` ノードを開き、ご自身のSlackワークスペースのCredential（認証）を設定し、通知を飛ばしたい `Channel ID` またはチャンネル名（例: `#sales-leads`）を指定します。

以上で設定は完了です！
これで、セミナー終了後に参加者がアンケートを送信した瞬間、あなたのSlackに「熱量スコア」と「完璧な営業メール」が次々と届くようになります。
