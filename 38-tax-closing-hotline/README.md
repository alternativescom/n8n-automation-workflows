# 38. Tax Closing Hotline & Document Organizer Bot (【決算対応】税理士ホットライン＆資料整理ボット)

⭐ **If you find this workflow helpful, please give it a Star! (役に立ったらぜひスターをお願いします！)** ⭐

## 🚀 概要 (Overview)
決算期にありがちな「税理士さんとの無駄なやり取り」や「提出書類の整理の手間」を、生成AIの力で劇的に削減する、実用性特化のワークフローです。

Slackの専用チャンネル（#決算ホットライン）に、社員がスマホで撮影した領収書の画像や、経費に関する質問を「雑投げ」するだけで、AI（Gemini）が意図を読み取り、以下の3つのルートに自動で振り分けます。

1. **不足情報あり（差し戻し）：** 「誰と行った交際費ですか？」など、前提条件が足りない質問には、AIが税理士の代わりにSlackで優しくヒアリングします。
2. **税理士へ連携（転送）：** 情報が完璧に揃った質問や判断が必要な書類は、AIが綺麗に要約して税理士のメールへ自動転送します。
3. **資料保管のみ：** 質問の必要がない単なる領収書などは、自動でリネームしてGoogleドライブの決算フォルダへ格納します。

現場の社員、経理担当者、そして税理士の「全員のストレス」をゼロにする、攻めのバックオフィスDXツールです。

## ✨ 特徴 (Features)
* **マルチモーダル解析:** Geminiの画像認識機能を活用し、写真で送られた領収書や税務署のハガキの内容を正確に読み取ります。
* **税理士への配慮（無駄なラリーの防止）:** 前提情報が欠けた質問をAIが水際でブロック（差し戻し）するため、税理士側の負担が激減し、良好な関係を保てます。
* **Switchノードによる美しく確実なルーティング:** AIが判定したステータス（`NEEDS_INFO`, `READY_TO_FORWARD`, `FILE_ONLY`）をもとに、n8nが正確に後続処理を分岐させます。

## ⚙️ 使用しているノード (Nodes Used)
1. **Slack Trigger:** 専用チャンネルへのメッセージ（テキスト・画像）投稿を検知。
2. **Google Gemini:** 投稿内容を解析し、3つのステータスのどれに該当するか判定。
3. **Code:** Geminiが出力したJSONをパースして後続に渡す。
4. **Switch:** ステータスに応じて処理を3つのルートに分岐。
5. **Slack:** ユーザーへの差し戻しヒアリング、または処理完了の通知。
6. **Gmail:** 税理士のメールアドレスへ、要約と添付ファイルを転送。
7. **Google Drive:** 決算資料を所定のフォルダへ自動保存。

## 🛠️ 使い方 (How to Use)
1. このディレクトリにある `workflow.json` をご自身のn8n環境にインポートします。
2. 各ノードの認証情報を設定します（Slack, Gemini, Gmail, Google Drive）。
3. **Slack Trigger** と各種 **Slack** ノードに、利用するチャンネル名（例：`#決算ホットライン`）を設定します。
4. **Gmail** ノードに、顧問税理士のメールアドレスを設定します。
5. ワークフローをActiveにして、Slackチャンネルに領収書の画像を投稿してみてください。
