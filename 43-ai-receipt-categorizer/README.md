# 43. AI Receipt Categorizer Bot (【経理DX】領収書・請求書のAI自動仕訳＆データ化ボット)

⭐ **If you find this workflow helpful, please give it a Star! (役に立ったらぜひスターをお願いします！)** ⭐

## 🚀 概要 (Overview)
「TaxHacker」のようなAI会計アプリの思想をn8nに組み込んだ、完全自動の仕訳＆データ化パイプラインです。

スマホのOCRアプリや複合機からテキスト化された「生の領収書データ（文字の羅列）」をWebhookに投げるだけで、AI（Gemini）が文脈を読み取ります。「日付」「金額」「支払先」の抽出はもちろん、自社独自のルールに基づいて**「勘定科目（消耗品費、会議費など）」を自動で推測し、会計ソフトやスプレッドシートにそのまま流し込める綺麗なJSONデータ（構造化データ）に変換**します。

## ✨ 特徴 (Features)
* **自社ルールに基づく高度な仕訳:** プロンプト内に「カフェで1人の場合は会議費」「AmazonのPC用品は消耗品費」といった自社特有のルールを記述することで、一般的な会計ソフトよりも実態に即した高精度な自動仕訳が可能です。
* **AIの「思考プロセス」の可視化:** 単に科目を割り当てるだけでなく、出力結果に `reasoning`（なぜその勘定科目にしたのかの推測理由）を含めることで、人間による最終確認やプロンプトのチューニングを容易にしています。
* **対話型から「完全自動化」への進化:** 人間が都度フォームから申請してAIのアドバイスを待つのではなく、裏側で大量のデータを黙々と処理するシステム間連携（API/Webhook）に特化しています。

## ⚙️ 使用しているノード (Nodes Used)
1. **Webhook:** OCRツールや他のアプリからの生のテキストデータを受信。
2. **Google Gemini:** プロンプトのルールに従ってテキストを解析し、構造化されたJSON形式で仕訳データを出力。
3. **Code (JSONパース):** Geminiの出力テキストから純粋なJSONオブジェクトを抽出。
4. **Slack:** 経理担当者へ自動仕訳の結果とAIの推測理由を通知（※Google Sheetsノード等に差し替えて直接記帳することも可能です）。

## 🛠️ 使い方 (How to Use)
1. このディレクトリにある `workflow.json` をご自身のn8n環境にインポートします。
2. 各ノードの認証情報を設定します（Gemini, Slack）。
3. **Slack** ノードで、通知先のチャンネル（例: `#経理・仕訳確認`）を指定します。
4. ワークフローをテスト状態（Listen for test event）にし、Postman等からWebhook宛てにテスト用の領収書テキスト（JSON）をPOSTして動作を確認してください。
