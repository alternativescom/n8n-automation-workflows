# 46. EC Price resaercher Bot (【EC支援】ライバルECサイトの価格変動＆新着AI監視ボット)

⭐ **If you find this workflow helpful, please give it a Star! (役に立ったらぜひスターをお願いします！)** ⭐

## 🚀 概要 (Overview)
競合他社のECサイトを毎日自動で監視し、「ゲリラセールによる値下げ」や「脅威となる新商品の投下」など、自社にとって重要な変化があった時**だけ**Chatworkにアラートを鳴らすAI監視ボットです。

従来のスクレイピングツールのように「HTMLの特定のタグ」を監視するのではなく、取得したサイトのテキストデータを丸ごとAI（Gemini）に読ませることで、「大決算セール！今だけ4,200円」といった**自然言語から「値下げ」という事実（意味）を正確に抽出して比較**します。

## ✨ 特徴 (Features)
* **デザイン変更に強いAI解析:** 相手サイトのレイアウトやCSSが変わっても、テキストの意味さえ取れればエラーにならず監視を継続できます。
* **通知疲れ（オオカミ少年化）の防止:** IFノード（条件分岐）を活用し、AIが「脅威となる変化あり」と判断した時のみ通知を飛ばします。変化がない日は何も通知しないため、現場の集中力を削ぎません。
* **推奨アクションの提示:** 単なる「値段が変わりました」という報告だけでなく、凄腕アナリストとして「自社も追従してクーポンを発行すべきか」等のアクションを提案します。

## ⚙️ 使用しているノード (Nodes Used)
1. **Schedule Trigger:** 毎朝10時など、指定した時間に監視をスタート。
2. **Set (Edit Fields):** テスト用の競合サイトデータ（昨日と今日）を生成（※実運用時はHTTP Requestやスプレッドシート等に置換します）。
3. **Google Gemini:** 2つのデータを比較し、意味を理解して脅威の有無を判定。
4. **Code (JSONパース):** Geminiの出力から純粋なJSONオブジェクトを抽出。
5. **IF (条件分岐):** 変化があった場合（`true`）と変化がない場合（`false`）でルートを分岐。
6. **Chatwork / NoOp:** 変化があった時だけChatworkへ警告を通知し、変化がなければ何もしない（NoOp）。

## 🛠️ 使い方と実運用への移行手順 (How to Use)

### 💡 まずはテスト実行で「AIの判定力」と「分岐」を体感する
本ワークフローには、すぐに威力をテストできるようダミーの「昨日のデータ」と「今日のテキスト（値下げ・新着あり）」を出力するSetノードが組み込まれています。
インポート後、そのまま「Test workflow」を実行し、**IFノードが「Trueルート」へ分岐してアラートが鳴る様子**を確認してください。

### ⚠️ 実運用環境へのカスタマイズ
実際の業務で使用する際は、ダミーデータを出力している「Setノード」を削除し、以下のようなノード構成に置き換えてください。
1. **昨日のデータ取得:** Google Sheetsノード等を使用し、前日に記録しておいた競合の価格データを読み込みます。
2. **本日のデータ取得:** HTTP Requestノードや、n8nのHTML Extractノード等を使用し、競合サイトから最新のテキストデータを取得します。
3. **データの更新:** 監視完了後、本日のデータを「明日のための比較用データ」としてスプレッドシート等に上書き保存するノードを追加します。
