# 29. AI Accounting Assistant (ベテラン経理チーフのAI領収書判定ボット)

## 🚀 概要
n8nの「Form Trigger」とGeminiの画像認識機能（Vision）を組み合わせた、AI経理アシスタントです。
スマホから「領収書の写真」と「申請理由（誰と・何のため等）」を入力して送信するだけで、Geminiが勤続20年の「ベテラン経理チーフ」として画像を解析します。
税務調査に耐えうるかどうかの一次判定、推奨される勘定科目、税理士を納得させるための「証拠の残し方」をプロフェッショナルなトーンでアドバイスします。

## ✨ 特徴
* **画像認識（マルチモーダル）対応:** 領収書に書かれた日付、金額、購入品目をAIが一瞬で読み取ります。スマホのカメラで撮影してそのままアップロード可能です。
* **コンプライアンスに配慮した設計:** 税理士法に抵触しないよう、断言を避け「顧問税理士への相談をスムーズにするための理論武装の支援」というスタンスをとっています。ジェンダーレスで実務的な「厳格な経理チーフ」のキャラクターを設定しています。
* **決算期・確定申告の工数削減:** 社内メンバーの経理リテラシー向上と、税理士とのコミュニケーションコスト（後からの確認作業）の大幅な削減に貢献します。

## ⚙️ 使用しているノード
* **n8n Form Trigger:** ユーザー入力用のWebフォーム（ファイルアップロード、テキスト入力）を提供。
* **Google Gemini:** 画像（Binary Data）とテキストを同時に受け取り、設定されたプロンプトに基づいて判定を出力。
* **Respond to Webhook:** 生成された判定テキストを、入力者のブラウザ画面にそのまま表示。

## 🚀 導入手順
1. 本リポジトリの `workflow.json` をn8nにインポートします。
2. `Analyze an image`（Gemini）ノードにご自身のGoogle Gemini API認証情報を設定します。
3. `Form Trigger` ノードを開き、「Test URL」または「Production URL」をスマホに送信して開きます。
4. スマホのカメラで手元の領収書を撮影し、申請理由を入力して送信テストを行ってください。
※APIの制限（429エラー）が出た場合も、n8n側のリトライ設定により自動で数回再実行されます。
