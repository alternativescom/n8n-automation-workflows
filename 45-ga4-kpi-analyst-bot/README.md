# 45. GA4 & Custom KPI Analyst Bot (【EC支援】GA4×自社KPI 凄腕ECアナリストボット)

⭐ **If you find this workflow helpful, please give it a Star! (役に立ったらぜひスターをお願いします！)** ⭐

## 🚀 概要 (Overview)
毎朝、前日のGoogle Analytics 4（GA4）のデータと、「自社独自のKPI目標」をAI（Gemini）に読み込ませ、ECサイトの改善点をChatworkに自動通知する凄腕アナリストボットです。

AIにただ数字を渡すだけでは「当たり前の要約」しか返ってきません。このワークフローの最大の特徴は、事前に**「カート遷移率の目標値」や「サイトのディレクトリ構造」といった『自社の文脈（コンテキスト）』をプロンプトに注入しておくこと**です。
これにより、AIが「カート遷移率が目標の60%に対して38%で未達です。購入画面での迷いが多い可能性があります」といった、プロのアナリストレベルのボトルネック特定と改善案を提示できるようになります。

## ✨ 特徴 (Features)
* **GA4 APIとの直接連携:** 毎朝決まった時間に、前日のセッション数やコンバージョン率などの最新データを自動取得します。
* **コンテキスト（文脈）の注入:** 一般論ではなく、「自社の目標値」に対するギャップから課題を抽出します。
* **Chatwork通知:** EC担当者が毎朝すぐに行動（アクション）に移せるよう、分かりやすくフォーマットしてチームのチャットへ届けます。

## ⚙️ 使用しているノード (Nodes Used)
1. **Schedule Trigger:** 毎朝9時など、指定した時間にワークフローを起動。
2. **Google Analytics (GA4):** 前日の主要なアクセスデータ（レポート）を取得。
3. **Set (Edit Fields):** テスト用のGA4データや自社KPIを生成（※実運用時は削除・置換します）。
4. **Google Gemini:** GA4データとKPIを突き合わせ、深い洞察に基づく分析レポートをJSONで出力。
5. **Code (JSONパース):** Geminiの出力から純粋なJSONオブジェクトを抽出。
6. **Chatwork:** 分析結果を見やすく装飾してチームへ通知。

## 🛠️ 使い方とGA4設定手順 (How to Use & GA4 Setup)

### 💡 なぜテストデータ（Setノード）を残しているのか？
本ワークフローには、あえてダミーのGA4データとKPIを出力する「Setノード」を組み込んでいます。
これは、**複雑なGA4のAPI連携設定を行う前に、「AIに自社の文脈（KPI）を教え込むと、どれほど鋭い分析レポートを返してくるか」をすぐにテスト実行して体感していただくため**です。

まずはそのままテスト実行（Test workflow）を行い、Geminiの素晴らしい分析能力を確認してください。

### ⚠️ 実運用への移行手順（テストデータの削除）
実運用（本番環境）で使用する際は、**ダミーデータを出力している「Setノード」を削除（または無効化）してください。**
そして、本物の「Google Analytics (GA4) ノード」から取得した実際の計測データが、Geminiノードへ渡るように接続をつなぎ直します。
（※自社のKPI目標については、別のSetノードを残してテキストで定義するか、Google Sheets等から読み込む構成に変更することをおすすめします）

### Step 1: Google Analytics 4 (GA4) の認証設定
n8nからGA4の本物のデータを取得するためには、Google Cloud Consoleでの設定が必要です。
1. [Google Cloud Console](https://console.cloud.google.com/) にアクセスし、プロジェクトを作成します。
2. 「APIとサービス」から **Google Analytics Data API** を有効化します。
3. 「認証情報」から **OAuth 2.0 クライアント ID** または **サービスアカウント** を作成します。
4. n8nのGA4ノードを開き、作成した認証情報を入力して接続を完了させます。

### Step 2: GA4ノードの取得条件設定
GA4ノードの「Get a report」設定にて、以下を指定します。
* **Property ID:** データを取得したいGA4のプロパティID（GA4管理画面で確認可能）。
* **Date Range:** `yesterday`（昨日）などを指定。
* **Metrics / Dimensions:** 取得したい指標（Sessions, Conversionsなど）を設定します。

### Step 3: AIと通知ノードの設定
1. **Geminiノード:** 認証情報を設定し、必要に応じてプロンプトのトーンを自社向けに調整します。
2. **Chatworkノード:** 通知先の `Room ID` と `API Token` を設定します。
