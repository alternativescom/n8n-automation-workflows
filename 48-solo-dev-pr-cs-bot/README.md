# 48. Solo Dev PR & CS Bot (【一人開発特化】広報・CS統括AIボット)

⭐ **If you find this workflow helpful, please give it a Star! (役に立ったらぜひスターをお願いします！)** ⭐

## 🚀 概要 (Overview)
一人開発者が抱える「ユーザー対応でのメンタル消耗」と「開発に夢中で広報（PR）がおろそかになる」という2つの大きな課題を、1つのn8nキャンバスで同時に解決する統合ボットです。

* **🛡️ CS（守り）ルート:** ユーザーからの辛辣な問い合わせをAIがマイルドな事象に変換して要約。お詫びの返信ドラフトと共に、開発者を励ましてくれます。
* **📣 PR（攻め）ルート:** 開発した機能一覧をAIに渡すだけで、誠実なリリースノートとSNS投稿用のドラフトを自動生成します。

## ✨ 特徴 (Features)
* **感情フィルタリング:** 怒りに満ちたクレームから「客観的なバグの事実」だけを抽出し、メンタルを守ります。
* **誠実なマーケティング:** 過度な煽りやバズ狙いではなく、開発者のこだわりが真っ直ぐ伝わる広報文を生成します。
* **一元管理:** 全く異なる2つの業務（CSと広報）を、1つのキャンバス内で並行して管理・稼働させることができます。

## ⚙️ 使用しているノード (Nodes Used)
* **Manual Trigger / Set:** テスト実行・ダミーデータ生成用
* **Google Gemini:** 感情分析＆マイルド化（CS）、リリースノート生成（PR）
* **Code:** JSONのパース処理
* **Slack:** チーム（自分）への通知とドラフトの共有

## 🛠️ 使い方と実運用への移行手順 (How to Use & Setup for Production)

このワークフローは、インポート直後は「テスト用」になっています。Manual TriggerをクリックしてAIの挙動を確認した後は、以下の手順で**あなたの実際のプロジェクトと連携（実運用化）**させてください。

### 🛡️ CSルートの実運用化（Webサイトからの問い合わせと連携）
ユーザーの問い合わせを受信したら自動で発火するように改修します。

1.  **Webhookノードの配置:**
    `Manual Trigger` と `Set (ダミーデータ)` を削除し、代わりに `Webhook` ノード（POSTメソッド）を配置します。
2.  **フォームとの連携:**
    WebhookのURLをコピーし、ご自身のWebサイトの問い合わせフォーム（Tally, Typeform, WordPressのContact Form 7など）の送信先（Webhook設定）に貼り付けます。
3.  **変数の紐付け:**
    Geminiノードを開き、`{{ $json.user_message }}` となっている部分を、Webhookから送られてきた実際の「問い合わせ内容」の変数（例：`{{ $json.body.message }}`）に書き換えます。

### 📣 PRルートの実運用化（タスク管理ツールとの連携）
週末やリリース日など、定期的に発火するように改修します。

1.  **Schedule Triggerノードの配置:**
    `Manual Trigger` を削除し、`Schedule Trigger`（例：毎週金曜日の17時など）を配置します。
2.  **Notion / GitHub等の連携:**
    `Set (ダミーデータ)` を削除し、代わりにタスクを管理しているツールのノード（`Notion` や `GitHub` 等）を配置します。
    *※Notionの場合は、ステータスが「完了」になったタスクだけを取得するようにFilterを設定します。*
3.  **タスク一覧のテキスト化:**
    取得した複数の完了タスクを1つの文章にまとめるため、ツールの直後に `Code` ノードを挟み、テキスト結合処理（第47弾参照）を行います。その結果をGeminiに渡すように変数を調整してください。

### 💬 Slackノードの設定
最後部にある2つのSlackノードを開き、ご自身のワークスペースのCredentialを設定し、通知を送りたい `Channel ID`（またはチャンネル名）を入力すれば完成です。
